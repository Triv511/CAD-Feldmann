Essential Git Commands

Master the most important Git commands every developer should know

Getting Started Commands

Initialize Repository

git init

Creates a new Git repository in the current directory. This is your first step when starting version control on a project.

Clone Repository

git clone [url]

Downloads a repository from a remote location (like GitHub) to your local machine, creating a complete copy with full history.

Configure Git

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Set up your identity for Git commits. This information appears in your commit history.

Basic Workflow Commands

The Add-Commit Cycle

üìä Check Status

git status

Shows which files are modified, staged, or untracked. Your best friend for understanding repository state.

‚ûï Stage Changes

git add [file]
git add .

Add files to the staging area. Use '.' to add all changed files, or specify individual files.

üíæ Commit Changes

git commit -m "Commit message"
git commit -am "Add and commit"

Save staged changes to repository history. The -am flag stages and commits tracked files in one step.

Branching Commands

üåø Branch Management

git branch
git branch [branch-name]
git branch -d [branch-name]

List branches, create new branches, or delete branches. Branches allow parallel development without affecting main code.

üîÑ Switch Branches

git checkout [branch-name]
git checkout -b [new-branch]
git switch [branch-name]

Move between branches or create and switch to new branches. Modern Git also includes 'switch' for clarity.

üîÄ Merge Branches

git merge [branch-name]
git merge --no-ff [branch-name]

Combine changes from different branches. The --no-ff flag preserves branch history for better tracking.

Remote Repository Commands

üåê Remote Management

git remote -v
git remote add origin [url]

View and manage remote repository connections. 'origin' is the conventional name for your main remote.

üì§ Push Changes

git push origin [branch-name]
git push -u origin [branch-name]

Upload your commits to remote repository. The -u flag sets up tracking for future pushes.

üì• Pull Changes

git pull origin [branch-name]
git fetch origin

Download changes from remote repository. Pull combines fetch and merge, while fetch only downloads.

History and Information Commands

üìú View History

git log
git log --oneline
git log --graph --all

Display commit history. Use flags for different formats: --oneline for compact view, --graph for visual branch structure.

üîç Show Differences

git diff
git diff --staged
git diff [commit1] [commit2]

Compare changes between files, commits, or branches. Essential for reviewing changes before committing.

üëÅÔ∏è Show Commit Details

git show [commit-hash]

Display detailed information about specific commits, including changes made. HEAD refers to the latest commit.

Undoing Changes

‚ö†Ô∏è Use These Commands Carefully

‚Ü©Ô∏è Undo Working Directory Changes

git checkout -- [file]
git restore [file]

Discard changes in working directory. This cannot be undone, so use with caution.

üì§ Unstage Files

git reset HEAD [file]
git restore --staged [file]

Remove files from staging area while keeping changes in working directory.

üîÑ Reset Commits

git reset --hard HEAD~1

Undo commits. --soft keeps changes staged, --hard discards everything. Use --hard very carefully!
